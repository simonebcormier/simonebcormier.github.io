Mohawk Canoe Journey

High Concept

Mohawk Canoe Journey is a 3D, first person virtual reality time travelling interactive experience based on the Akwesasne Mohawk Journey. Experience a journey through a canoe ride down a river which takes you to three different time periods of the Akwesasne Mohawk people; pre-contact, post-contact and present.

Project Info

  • Project: Carleton University Team Project
  • Role: Gameplay Programmer, UI Programmer
  • Web Framework: A-Frame
  • Programming Languages: HTML, CSS, Javascript

Responsibilities

  • Gameplay implemetation
  • UI implementation
  • 3D model importing
  • Playable on keyboard, mobile and VR
  • Prototyping and iteration
  • Communication with clients to bring vision to life

Gameplay Implementation

  • Made player movement work for all devices- WASD for keyboard movement, teleport pads for touchscreen and VR
  • When player interacts with an object in the world that is interactable, they can either view it to see it in full screen and focus on it, fully rotate it around by clicking and dragging and look at it, or pick it up which puts it into their canoe for the rest of the game. Picking up the item and looking at it was important as the clients sent us reference photos for the items we put in game of objects that were important to them and we wanted players to pay attention to them and really see them fully
  • Player uses canoe to travel between the three levels, at the end it loops around

UI Implementation

  • Start menu, end menu implementation
  • A big part of our game was to bring the player on a journey to learn about the Akwesasne Mohawk people, so UI was essential in teaching the player key moments of the journey
  • I implemented the item interaction menu to give the player options to pickup the item, view it, see more information or close the menu. Buttons have hover states and click states
  • Text in the UI is made for resizing since the game was made for monitors, mobile and VR so text sizing for different devices was essential
  • Paired with the UI, when you pick up an item I implemented audio clips of clients saying the names of the objects to make it even more personal for them and informative for the players
