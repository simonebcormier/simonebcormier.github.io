Space Shooter Prototype

Project Overview

I developed a deck-building card game prototype in Unity, focusing on creating engaging card-based mechanics and a user-friendly interface. The project involved designing and implementing core systems such as card management, deck construction, and interactive UI elements. I utilized Unity's prefab and scripting systems to rapidly prototype features, iterate on gameplay, and polish the user experience based on feedback. This project demonstrates my ability to design, implement, and refine game systems using C# and Unity, as well as my skills in rapid prototyping and user-centered design.

Project Info

  • Project: VFS Solo Project
  • Role: Programmer, UI Programmer
  • Engine: Unity
  • Programming Languages: C#

Responsibilities

  • Designed and implemented the card management system and deck construction
  • Developed interactive UI for deck building and card selection using Unity's UI tools
  • Integrated visual assets and animations to enhance user experience
  • Implemented data structures for efficient card and deck handling
  • Prototyped and iterated on gameplay mechanics based on playtesting and feedback
  • Conducted playtesting and analyzed user feedback to refine game systems

Interactive UI

  • Utilized Unity's Canvas system to create a responsive, layered UI for deck building and card selection.
  • Implemented intuitive drag-and-drop and button controls for adding/removing cards using Unity's UI components.
  • Provided feedback on deck size, stats, and card limits.
  • Used clear icons, tooltips, and section titles to reduce confusion and guide users.
  • Enabled deck personalization features like renaming and detailed card info popups, all managed through Unity's UI system.

Prototyping and Iteration

  • Researched leading card games to inform UI and feature design.
  • Built and tested three prototypes, each improved through user feedback.
  • Simplified visuals, clarified controls, and added real-time deck/stat indicators.
  • Enhanced usability by addressing tester pain points in each iteration.
  • Validated improvements through repeated playtesting and refinement.
